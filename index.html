2nd Year Project

Hotel Management System

As part of a 2nd year project containing four members we developed a hotel management system that had 2 kinds of users, Administrators and people who wanted to book a room in the hotel and 3 kinds of rooms which where single, double and twin. Together we created a java swing application that had a JDBC connection to an SQL database. We researched other similar systems and implemented the best features of those systems into our own. We drew storyboards of the system detailing what each screen should allow a user to do. Each user of the system had an account which they could change the details of and each users password was hashed and stored in our SQL database.

We decided that people trying to book a room didn't have to sign in right away as this would annoy users. This allowed users to browse available rooms before proceeding with a booking. When a user was ready to proceed with their booking then they would be asked to sign in so they could complete their booking. When a user was proceeding with their booking they were asked for their credit card details. Credit card numbers were validated using the Luhn Algorithm but they were not stored.

Administrators could add, update and delete rooms in the hotel, bookings on rooms and the extras the hotel offered such as a round of golf. As well as the basic CRUD functionality administrators could print reports on the hotel. Reports such as a pie chart showing which rooms were the most popular in the hotel, the amount of revenue the hotel made for a particular year which had a month by month break down of the bookings on particular rooms or the breakdown of a particular month.

This project gave us a great insight into the development of a system and each of us benefited greatly from this as the project group received a grade of 94 which was the highest grade in the year.

3rd Year Project

Android Application used to discover local restaurants

As part of a 3rd year project consisting of two team members, we decided to create an android application which allowed users of the application to find restaurants closest to them and a .NET web application that allowed restaurants to sing up to our service and be in charge of what users saw about them in our application. The android application allowed users to firstly, find a restaurant that was closest to them, secondly, allow a user to book a table in that restaurant as if they were booking a seat in the cinema while the .NET application removed the hassle involved for restaurants in trying to make their own application for their customers. This was done by allowing restaurants to enter information about their restaurant such as their location, restaurant image and their floor plan.

The android application was designed to allow users to find restaurants that were closest to them based on their current location. However their search criteria wasn't limited to this as they could also search for restaurants based on the options we provided them with such as searching by name or searching for wheelchair accessible restaurants. Each restaurant presented to the user in the application had their own section purely about their restaurant which provided users with useful information. This information was entered by the restaurants owners through the .NET application.

Booking a table at a restaurant was the key functionality to this project and it proved to be a quite a challenge. Myself and my project partner wanted the booking of a table to be easy like booking a cinema ticket for a film you want to see. With that in mind we decided to allow a user to book a table by simply selecting the table they wanted from a floor plan that was designed by the restaurant themselves. Once a user had confirmed their booking the user would receive an email and a reminder was automatically set on the android device.

A web application was developed along with the android application to allow restaurants to be in charge of what users of the android application could see. We kept track of restaurants location by asking them to show us on a google map where their restaurant was located and in the background we saved the longitude and latitude of the location so we could use it to determine how close restaurants were to users using the harversine algorithm (an algorithm used to determine the distance between to points). The most important functionality of the web application was allowing each restaurant create one or more floor plans as this was how users made bookings on restaurants.

4th Year Project

Home Security Application

My fourth year project is going to be an android application that allows users to turn their old android devices into security cameras. Users will control their old android devices by sending commands to them from their day to day device. Commands will be sent using Azures notification hubs, which will communicate with androids PNS (Push Notification Service), Google Cloud Messaging. Commands such as enabling/disabling motion detection and turning on or off the flash light on the device will be available to users.

Upon detected motion the user will receive a picture showing the cause of the detected motion in an email. Users can then choose from a list of their available security cameras to view the live feed coming from these devices. Live video feeds will be available from Azure Media Services which will be responsible for producing multiple bit rates for a video feed. This allows for different quality of videos to be streamed thus allowing videos to be available to more users.
